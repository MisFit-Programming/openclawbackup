# Memory Maintenance Operating Rules

## Purpose

Keep Dobbie useful, consistent, and light-footed by curating memory instead of hoarding it.

## Core principles

- Prefer curation over accumulation.
- Preserve decisions, preferences, workflows, lessons, and standing tasks.
- Skip routine noise, one-off chatter, and already-obvious operational fluff.
- Do not copy sensitive personal details into broader/shared contexts.
- Treat daily memory files as the primary operational journal.
- Treat long-term memory as a curated summary, not a transcript dump.

## What belongs in daily memory

Keep in `memory/YYYY-MM-DD.md`:
- events from the day
- notable requests
- workflow changes
- one-off incidents
- successful or failed experiments
- new automations, cron jobs, skills, integrations, and configuration changes

## What may belong in long-term memory

Only distill into long-term memory when appropriate and privacy-safe:
- durable user preferences
- stable persona/tone guidance
- important decisions
- recurring workflows
- enduring project context
- repeated lessons worth preserving

## What should usually be pruned or left out

- casual banter
- duplicate facts repeated across multiple daily files
- temporary glitches already resolved with no lesson attached
- large raw logs
- secrets or credentials
- information that would be risky to surface broadly

## Privacy boundary

Be conservative with long-term memory curation when working in shared/group contexts.
If the information is private, personal, credential-adjacent, or not clearly meant for broader recall, leave it in daily notes or skip it entirely.

## Signs memory needs maintenance

- repeated facts show up across several daily files
- old workflows no longer match reality
- standing automations changed but memory does not reflect it
- too much raw noise makes useful recall harder
- important user preferences are only present in transient daily notes

## Good maintenance outcomes

- fewer duplicates
- cleaner daily notes
- stronger continuity across sessions
- better recall of durable preferences and workflows
- less clutter and less risk of inappropriate memory bleed
